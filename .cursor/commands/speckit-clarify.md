# Spec-Kit Clarify

Help clarify ambiguities in spec or design documents.

## References
- @.specify/memory/constitution.md
- Current spec.md or plan.md in context

## Clarification Focus

### 1. Requirements Clarification
- Are business rules clear?
- Are user stories complete?
- Are acceptance criteria testable?
- Are edge cases covered?

### 2. Technical Design Clarification
- Are API interface definitions clear?
- Is the data model complete?
- Are dependencies clear?
- Is the error handling strategy clear?

### 3. Task Breakdown Clarification
- Can tasks be implemented independently?
- Are dependencies correct?
- Are acceptance criteria clear?
- Is the effort estimation reasonable?

## Output Format

For each ambiguity:
1. Point out specific location
2. Explain why it's ambiguous
3. Provide possible interpretations or suggestions
4. Pose questions that need confirmation
